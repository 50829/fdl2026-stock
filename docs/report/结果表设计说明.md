# 报告结果表设计说明

日期：2026-06-11

## 1. 总原则

实验报告里的结果表不应该把仓库里所有实验都堆进去。正式报告必须先满足作业 PDF 的要求：数据处理、深度学习模型、训练收敛、IC/ICIR、历史回测、模拟交易分析。其他研究型实验可以放在附录或开发记录中。

主报告建议保留 8 类表：

1. 数据集与切分表。
2. 数据预处理与特征表。
3. 深度学习模型结构表。
4. 训练收敛表。
5. 模型主结果表。
6. 策略主结果表。
7. 模拟交易记录表。
8. 最终候选策略对比表。

## 2. 数据集与切分表

这张表放在数据章节，用来说明实验时点是否安全。

建议字段：

| 字段 | 说明 |
| --- | --- |
| 阶段 | 训练集、验证集、测试集、walk-forward |
| 日期范围 | 开始日期和结束日期 |
| 样本行数 | 股票-日期样本数量 |
| 股票池口径 | 是否启用 universe 和流动性过滤 |
| 标签 | 使用 `label1d` 或 `label5d` |
| 用途 | 训练、调参、最终评估、样本外验证 |

这张表的作用是防止报告被质疑“是否用了未来数据”。

## 2.1 数据预处理与特征表

这张表对应 PDF 中“数据预处理规范”和“特征工程”的要求。

建议字段：

| 字段 | 说明 |
| --- | --- |
| 处理步骤 | 缺失值处理、股票池过滤、标准化、标签构造等 |
| 输入数据 | 原始量价、基本面、资金流等 |
| 输出文件 | 对应 processed 文件 |
| 是否使用未来信息 | 必须明确为否 |
| 备注 | 说明边界情况 |

## 2.2 深度学习模型结构表

PDF 明确要求流程中必须包含深度学习模型，所以这张表必须进入正文。

建议字段：

| 模型 | 输入 | 结构 | 输出 | 作用 |
| --- | --- | --- | --- | --- |
| MLP | 截面特征 | 多层全连接 | 收益排名分数 | 基础神经网络 |
| GRU | 历史窗口序列 | 循环神经网络 | 收益排名分数 | 捕捉时序依赖 |
| TCN | 历史窗口序列 | 时间卷积 | 收益排名分数 | 捕捉局部模式 |
| residual-rank MLP | 树模型预测和排名 | MLP | 融合分数 | 神经网络融合 |

这张表的目的不是证明神经网络最好，而是证明模型设计过程满足课程要求。

## 2.3 训练收敛表

PDF 要求说明训练有效、损失能够收敛。建议放一张简表，配一张损失曲线图。

建议字段：

| 模型 | 训练轮数 | 初始训练损失 | 最终训练损失 | 最优验证损失 | 是否收敛 |
| --- | ---: | ---: | ---: | ---: | --- |

如果某些早期实验没有完整曲线，就放 summary 中可恢复的训练日志，并在报告中说明。

| XGBoost | label1d | 40 | 验证集 |  |  |  |  |  |

不建议把所有深度模型和所有中间调参实验都放进主表。深度模型可以做一张附录表，正文只写“探索过但没有稳定超过树模型”。

## 4. 策略主结果表

这张表是报告最重要的表之一。它回答“模型分数到底能不能赚钱”。

建议字段：

| 字段 | 说明 |
| --- | --- |
| 数据集 | 验证集或测试集 |
| 模型 | `label1d LightGBM`、`label5d final` 等 |
| 策略 | `topk20_drop3`、`rank buffer`、`rolling_p10_h5` 等 |
| 总收益 | 回测区间总收益 |
| 年化收益 | 便于不同区间比较 |
| 夏普 | 风险调整后收益 |
| 最大回撤 | 组合风险 |
| 平均换手 | 成本敏感性 |
| 平均持仓数 | 策略容量与集中度 |
| 成本假设 | 例如 5 bps |

主表不要超过 8 行。建议只保留：

1. `label1d LightGBM + topk20_drop3`
2. `label1d LightGBM + rank buffer`
3. `label5d final + rolling_p10_h5`
4. `label5d final + rolling_p20_h3`
5. 指数基准
6. 等权股票池基准

## 5. walk-forward 稳定性表

这张表说明结果不是只在一个测试集上好看。

建议字段：

| 字段 | 说明 |
| --- | --- |
| 特征组 | `all_windows`、`no_20d`、`short_5_10` |
| 平均 IC | 多年平均信号质量 |
| 最低 IC | 最差年份信号是否仍为正 |
| 平均 ICIR | 多年信号稳定性 |
| 最低 ICIR | 最差年份稳定性 |
| 平均夏普 | 多年策略表现 |
| 最低夏普 | 最差年份表现 |
| 最深回撤 | 风险底线 |
| 平均换手 | 成本压力 |

这张表应该配一张样本外净值曲线图，正文里不要只放数字。

## 6. 2024 压力测试表

这张表专门解释为什么很多策略都有大约 40% 回撤。

建议字段：

| 策略 | 2024 IC | 2024 ICIR | 2024 收益 | 2024 最大回撤 | 平均换手 | 解释 |
| --- | ---: | ---: | ---: | ---: | ---: | --- |
| 普通 topk |  |  |  |  |  | 信号仍有效，但组合路径承压 |
| rolling 策略 |  |  |  |  |  | 换仓较慢，仍受市场冲击 |
| rank buffer |  |  |  |  |  | 降低换手但不能完全避险 |
| defensive 策略 |  |  |  |  |  | 回撤下降，收益也下降 |

这张表的重点不是证明某个策略最好，而是解释“为什么模型 IC 正，但组合仍然大回撤”。

## 7. 成本和滑点敏感性表

如果篇幅允许，建议放一张成本敏感性表。

建议字段：

| 策略 | 成本 bps | 滑点 bps | 总收益 | 夏普 | 最大回撤 | 平均换手 |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |

这张表可以说明高换手策略是否过度依赖低成本假设。特别是 `label1d` 策略，必须展示成本敏感性，因为它的收益优势来自更频繁的换仓。

## 8. 最终候选策略对比表

最后结论章节建议放一张最终候选表。

建议字段：

| 候选 | 模型 | 策略 | 优点 | 风险 | 当前定位 |
| --- | --- | --- | --- | --- | --- |
| 稳健候选 | label5d final | rolling_p10_h5 | 回撤较低，路径平滑 | 收益弹性较弱 | 保守基线 |
| 进攻候选 | label1d LightGBM | topk20_drop3 | 收益强，信号敏感 | 换手和回撤较高 | 研究候选 |
| 折中候选 | label1d LightGBM | rank buffer | 收益强，减少部分无效换手 | 仍需风险约束 | 后续主线 |
| 防守候选 | label1d 或 label5d | defensive rank buffer | 压力期回撤低 | 收益牺牲大 | 风控模块 |

这张表负责把实验结果转化为清晰结论。

## 8.1 模拟交易记录表

PDF 要求记录并分析模拟交易收益，而且要求模拟交易操作与深度学习模型结果一致。

建议字段：

| 日期 | 模型推荐股票 | 实际买入 | 实际卖出 | 收盘持仓 | 当日收益 | 是否按模型执行 | 偏离原因 |
| --- | --- | --- | --- | --- | ---: | --- | --- |

这张表需要结合同花顺模拟交易截图手工补齐。仓库只能提供模型推荐结果，不能自动知道手机端最终成交记录。

## 9. 图表配套

报告中建议最少放 6 张图：

1. 神经网络训练损失曲线。
2. 模型 IC 或 ICIR 对比柱状图。
3. `label1d` 与 `label5d` 代表策略净值曲线。
4. walk-forward 样本外净值曲线。
5. 2024 压力窗口净值或回撤曲线。
6. 同花顺模拟交易收益趋势截图。

## 10. 主报告不建议放的内容

以下内容可以放附录，不建议占正文篇幅：

1. 所有中间失败模型。
2. 所有调参组合。
3. 每天手工查询股票排名的明细。
4. 无流动性过滤的诊断排名。
5. 尚未完成的无流动性模型训练。
6. 过多单策略单模型的小图。
7. 与作业主题无关的仓库重构细节。

正文要围绕作业要求展开：我们如何用深度学习方法完成股票趋势预测，如何避免数据泄露，如何把模型分数转化为交易策略，以及模拟交易中学到了什么。
