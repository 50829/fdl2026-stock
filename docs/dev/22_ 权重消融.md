我查了一下相关资料。这个 NS-NTK 思路对我们最有价值的地方不是直接搬 PVR/EAT，而是把它拆成三个可验证问题：

  1. 历史数据是不是会过期？
  2. 深度模型的随机训练噪声能不能被平均掉？
  3. 预测分数太抖导致换手高，能不能通过平滑提高净收益？

  相关背景里，漂移分布学习不是新问题。Mohri 和 Muñoz Medina 2012 年就用 discrepancy 分析 drifting distributions，并讨论如
  何用历史数据；近年也有 unknown drift 下自适应学习的 NeurIPS 工作。EMA/SWA 也有成熟依据，EMA 权重平均被系统研究为降低
  SGD 噪声、改善鲁棒性，SWA 则是通过平均 SGD 轨迹找到更平坦解。NS-NTK 仓库本身公开在 GitHub，确实包含 PVR/EAT 官方实现。
  参考：NS-NTK GitHub、Mohri drifting distributions、adaptive unknown drift、EMA、SWA。

  最建议做的实验

  1. 指数时间衰减样本权重

  这是最直接对应“历史数据有保质期”的实验。

  做法：

   版本            训练样本权重
  ━━━━━━━━━━━━━━  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   uniform         所有训练样本权重相同
  ──────────────  ───────────────────────────────────────
   half-life 1y    越近样本权重越高，半衰期约 252 交易日
  ──────────────  ───────────────────────────────────────
   half-life 2y    半衰期约 504 交易日
  ──────────────  ───────────────────────────────────────
   rolling 1y      只用最近 1 年
  ──────────────  ───────────────────────────────────────
   rolling 2y      只用最近 2 年

  优先跑 label1d LightGBM/XGBoost，再跑一个深度模型比如 MLP 或 GRU。
  看指标：valid/test IC、ICIR、topk10_drop2 收益、最大回撤、换手。

  这会成为报告里很强的一节：理论说“漂移快就要更快忘记历史”，我们实证检验 A 股日频到底适不适合遗忘。

  2. 深度模型 EMA 权重平均

  对应 NS-NTK 里的“随机方差项太大”。不直接做 PVR，先做更稳的 EMA。

  做法：

  训练 MLP/GRU/TCN 时同时维护几套 EMA 权重：

  ema_decay = 0.99 / 0.995 / 0.999

  每个 epoch 保存：

   模型版本           说明
  ━━━━━━━━━━━━━━━━━  ━━━━━━━━━━━━━━━━━━━━━━━━━
   last checkpoint    普通最后权重
  ─────────────────  ─────────────────────────
   best valid loss    当前已有 early stopping
  ─────────────────  ─────────────────────────
   EMA 0.99           更短窗口平均
  ─────────────────  ─────────────────────────
   EMA 0.995          中等平滑
  ─────────────────  ─────────────────────────
   EMA 0.999          长窗口平滑

  看 EMA 是否提高 ICIR、降低 prediction volatility。
  这比改优化器靠谱，工程量也小。

  3. 多种子深度模型 ensemble

  我们现在深度模型单次训练容易受随机性影响。可以跑：

  seed = 2021, 2022, 2023, 2024, 2025

  对 MLP/GRU/TCN 的预测取平均，再比较：

   版本                   目的
  ━━━━━━━━━━━━━━━━━━━━━  ━━━━━━━━━━━━━━━━━━━━━━━━━━
   single seed            当前基线
  ─────────────────────  ──────────────────────────
   3 seed ensemble        方差缩减
  ─────────────────────  ──────────────────────────
   5 seed ensemble        更强方差缩减
  ─────────────────────  ──────────────────────────
   EMA + seed ensemble    同时做权重平均和模型平均

  如果深度模型本身 IC 不高，但 ensemble 后 ICIR 明显提升，这就能支持“深度模型有弱信号，但需要降噪”的结论。

  4. 预测分数时间平滑

  对应 PVR 的实际效果：预测更平滑、换手更低、扣费后收益更好。

  做法不是改梯度，而是直接对每日分数平滑：

  score'_t = alpha * score_t + (1-alpha) * score'_{t-1}

  扫描：

  alpha = 1.0, 0.8, 0.6, 0.4

  分别对：

   模型                         策略
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━  ━━━━━━━━━━━━━━━━━
   label1d LGB+XGB rank_mean    topk10_drop2
  ───────────────────────────  ─────────────────
   label1d LGB+XGB rank_mean    rank buffer
  ───────────────────────────  ─────────────────
   label5d LGB/XGB              rolling tranche

  看净收益、换手、最大回撤。
  这个实验非常可能有用，因为我们目前策略收益对换手和流动性很敏感。

  第二优先级实验
  5. 月度/年度 IC 稳定性图

  不要只报全期 IC。按月、按年画：

   图                      作用
  ━━━━━━━━━━━━━━━━━━━━━━  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   月度 IC 折线            看信号什么时候失效
  ──────────────────────  ─────────────────────────────────
   年度 ICIR 柱状图        看稳定性
  ──────────────────────  ─────────────────────────────────
   月度 topk 收益热力图    看策略是否集中赚某几个月
  ──────────────────────  ─────────────────────────────────
   模型分歧度              LGB 和 XGB 排名差异是否预示风险

  这能回应“市场漂移速度”问题：如果某些年份 IC 明显变差，说明历史关系在漂移。

  6. 训练窗口 walk-forward 重做

  之前已经做过窗口消融，但可以扩展成更系统版本：

   训练窗口             验证年份
  ━━━━━━━━━━━━━━━━━━━  ━━━━━━━━━━━
   expanding            2021-2026
  ───────────────────  ───────────
   rolling 1y           2021-2026
  ───────────────────  ───────────
   rolling 2y           2021-2026
  ───────────────────  ───────────
   rolling 3y           2021-2026
  ───────────────────  ───────────
   exponential decay    2021-2026

  这个是 NS-NTK Theorem 1 最对应的实验。

  7. 深度模型和 GBDT 的残差互补性

  我们现在发现直接融合深度模型会拖累。但可以更细地看：

  ──────────────────────────────────────  ────────────────────────────
   deep score smoothing 后是否改善         未平滑前可能太吵

  结论可能不是“深度模型进主分数”，而是“深度模型做确认信号”。
  不建议直接把 PVR/EAT 搬进主实验。原因：

  1. PVR 需要在线估计梯度主方差方向，超参多，容易跑不稳。
  2. EAT 需要 NTK 或近似 kernel preconditioner，工程复杂。
  3. 我们当前 deadline 虽然延期到 6 月 30 日，但更值得做的是可解释、可复现、能放进报告表格的实验。
  4. PVR/EAT 真正要对标的是 AdamW、SWA、EMA、Shampoo/K-FAC 等优化器，不是简单跑一下就能证明有效。

  我的建议排序
  最值得做的 5 个：

  4. 预测分数时间平滑：直接看换手和净收益。
  5. 月度/年度 IC 稳定性分析：作为报告核心图表。

  这样报告可以形成一条很清楚的主线：

  金融数据非平稳
  => 历史样本要做时间加权
  => 深度模型训练噪声要做 EMA / ensemble 降噪
  => 预测分数要做时间平滑降低换手
  => 最终用 ICIR、回撤、换手而不是单纯 IC 判断改进

  这比直接说“我们用了某个复杂优化器”更扎实，也更符合你们现在项目已经有的模型和回测框架。
