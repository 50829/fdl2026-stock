# 16 每日预测与交易生成流程

日期：2026-06-09

## 1. 文档目的

本文档记录每日用最新数据生成股票预测和交易候选的标准流程。

它主要回答：

1. 每日原始数据应该放在哪里。
2. `decision_date` 和 `trade_date` 应该怎么理解。
3. 当前线上 `label1d` 模型应该怎样跑。
4. 输出文件应该怎么命名、怎么解释。
5. 哪些结果可以用于正式交易，哪些只能用于诊断。

## 2. 日期口径

每日预测必须明确两个日期：

| 字段 | 含义 | 示例 |
| --- | --- | --- |
| `decision_date` | 生成信号所使用的最后一个已知交易日 | `20260605` |
| `trade_date` | 计划执行交易的交易日 | `20260608` |

例子：

```text
2026-06-05 收盘后拿到 6 月 5 日数据
2026-06-06 和 2026-06-07 是周末
下一次交易日是 2026-06-08

因此：
decision_date = 20260605
trade_date = 20260608
```

不能默认简单使用自然日加一天。遇到周末、节假日时，必须显式传入 `--trade-date`。

## 3. 原始数据要求

每日数据包放在：

```text
data/raw/
```

推荐文件名：

```text
data/raw/YYYYMMDD.zip
```

例如：

```text
data/raw/20260605.zip
```

压缩包内至少应该包含：

```text
daily YYYYMMDD.csv
metric YYYYMMDD.csv
moneyflow YYYYMMDD.csv
stock st YYYYMMDD.csv
daily open TRADE_DATE.csv
```

以 `20260605 -> 20260608` 为例，应包含：

```text
daily 20260605.csv
metric 20260605.csv
moneyflow 20260605.csv
stock st 20260605.csv
daily open 20260608.csv
```

各文件用途：

| 文件 | 用途 |
| --- | --- |
| `daily` | 日频开高低收、成交量、成交额、均价 |
| `metric` | 估值、市值、换手率等指标 |
| `moneyflow` | 大单、小单、主力资金流等字段 |
| `stock st` | 当日 ST 股票列表 |
| `daily open` | 交易日开盘价和前收盘价，用于生成交易参考 |

如果缺少 `daily open`，模型仍可计算预测分数，但无法在输出里给出交易日开盘参考价。

## 4. 当前正式模型口径

当前每日 `label1d` 正式口径使用：

```text
模型：label1d_lightgbm_top40_alpha0
目标：label_1d__cs_rank
基础模型：LightGBM top40
融合 alpha：0.0
```

模型路径：

```text
outputs/models/20260601_000000__label1d_top40/lightgbm/model.txt
```

同时传入 XGBoost 和 residual-rank MLP 文件，是因为当前 `live-rank` 管道仍保留融合模型入口；但 `alpha=0.0` 时：

```text
final_pred = pred_lgb
```

因此每日 `label1d` 分数实际就是 LightGBM 的 `label_1d__cs_rank` 预测分数。

重要说明：

```text
final_pred 不是百分比收益率。
final_pred 是截面排序分数。
```

分数越高，表示模型认为该股票下一交易周期相对其他候选股票更靠前。不能把 `0.15` 解释成预测收益 `15%`。

## 5. 标准运行命令

以 `20260605 -> 20260608` 为例：

```bash
conda run -n fdl python -m src.experiments live-rank \
  --decision-date 20260605 \
  --trade-date 20260608 \
  --artifact-registry configs/registry/artifacts.yaml \
  --lgb-model outputs/models/20260601_000000__label1d_top40/lightgbm/model.txt \
  --xgb-model outputs/models/20260601_000000__label1d_top40/xgboost/model.json \
  --fusion-model outputs/models/20260601_000000__label1d_fusion_top40/residual_rank_mlp/residual_rank_mlp.pt \
  --alpha 0.0 \
  --model-name label1d_lightgbm_top40_alpha0 \
  --out-dir outputs/live/20260608__label1d_lgb_top10__from_20260605
```

运行环境：

```text
conda 环境：fdl
```

## 6. 输出目录规范

每日预测输出目录采用：

```text
outputs/live/TRADE_DATE__实验名称__from_DECISION_DATE
```

例如：

```text
outputs/live/20260608__label1d_lgb_top10__from_20260605
```

命名要求：

1. 日期放在最前面。
2. 实验名称必须说明模型和用途。
3. `from_YYYYMMDD` 必须保留，表示使用哪一天的数据决策。
4. 目录名不能包含人名、临时缩写、含糊描述。

## 7. 主要输出文件

标准输出包括：

| 文件 | 含义 |
| --- | --- |
| `summary.json` | 本次预测摘要 |
| `run_meta.json` | 命令参数、输入产物、注册表和运行元数据 |
| `live_features_DECISION_DATE.parquet` | 当日 live 特征 |
| `live_predictions_DECISION_DATE.csv` | 全候选股票预测分数 |
| `top10_DECISION_DATE_for_TRADE_DATE.csv` | 前 10 名交易候选 |
| `mentioned_stock_ranks_TRADE_DATE.csv` | 默认关注股票列表排名 |

如果临时查询指定股票，可以额外输出：

```text
requested_stock_scores_label1d_TRADE_DATE.csv
```

## 8. 正式预测与诊断预测

正式预测必须使用和训练一致的股票池口径：

```text
in_universe = 基础可交易条件 + 流动性过滤
```

正式交易只应使用：

```text
live_predictions_DECISION_DATE.csv
top10_DECISION_DATE_for_TRADE_DATE.csv
```

如果为了研究某只股票为什么没有排名，可以临时跑诊断版本，例如：

```text
取消流动性过滤
放宽候选池
查看被排除原因
```

但诊断结果不能直接和正式预测分数比较，也不能直接作为正式交易建议。

原因是模型特征里有大量截面排名和截面标准化特征。股票池改变后，特征值本身会改变，最终分数也会改变。

## 9. 指定股票查询流程

如果用户指定一批股票，标准处理方式是：

1. 先跑正式 `live-rank`。
2. 从 `live_predictions_DECISION_DATE.csv` 中按 `ts_code` 匹配。
3. 如果没有匹配到，检查候选池排除原因。
4. 明确说明没有分数的原因，而不是简单写“无”。

常见原因：

```text
原始数据缺失
ST
北交所
上市天数不足
当日无成交
流动性过滤
特征构造后不在候选池
```

## 10. 每日检查清单

运行前检查：

```text
data/raw/YYYYMMDD.zip 是否存在
压缩包内是否包含 daily、metric、moneyflow、stock st
是否有下一交易日 daily open 文件
trade_date 是否考虑周末和节假日
模型路径是否仍是当前正式模型
输出目录是否符合命名规范
```

运行后检查：

```text
summary.json 是否生成
candidate_count 是否异常变化
top10 文件是否生成
指定股票是否能匹配到排名
是否有股票因为过滤条件被排除
run_meta.json 是否记录了完整参数
```

如果 `candidate_count` 明显异常，必须先排查数据和股票池口径，不能直接使用结果。
