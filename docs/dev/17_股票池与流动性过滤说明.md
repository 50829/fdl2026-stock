# 17 股票池与流动性过滤说明

日期：2026-06-09

## 1. 文档目的

本文档说明当前项目中股票池、流动性过滤、训练口径和每日预测口径之间的关系。

重点结论：

```text
当前 label1d 模型是使用带流动性过滤的股票池训练出来的。
正式预测也必须保留同样的流动性过滤。
取消流动性过滤只能用于诊断，不能直接作为正式交易结果。
```

## 2. 当前股票池定义

当前股票池字段是：

```text
in_universe
```

它在预处理和每日 live 预测中都用于决定哪些股票进入特征构造和模型预测。

当前逻辑可以概括为：

```text
in_universe =
  非北交所
  且 非 ST
  且 当日成交量大于 0
  且 当日成交额大于 0
  且 上市天数满足最低要求
  且 通过流动性过滤
```

配置文件：

```text
configs/config.yaml
```

关键配置：

```yaml
universe:
  min_list_days: 60
  liquidity_filter:
    enabled: true
    window: 20
    bottom_pct: 0.2
```

含义：

| 配置 | 含义 |
| --- | --- |
| `min_list_days: 60` | 至少上市 60 天 |
| `enabled: true` | 启用流动性过滤 |
| `window: 20` | 使用近 20 个交易日成交额均值 |
| `bottom_pct: 0.2` | 剔除底部 20% |

## 3. 流动性过滤是什么

流动性过滤使用字段：

```text
amount_mean_20
```

它表示近 20 个交易日的成交额均值。

每天在基础可交易池内，对 `amount_mean_20` 做截面排名，然后剔除底部 20%：

```text
passes_liquidity = amount_mean_20 的当日分位数 > 20%
```

最终：

```text
in_universe = base_mask & passes_liquidity
```

其中 `base_mask` 是非北交所、非 ST、有成交、上市天数足够等基础条件。

## 4. 为什么要保留流动性过滤

流动性过滤不是为了让模型指标更好看，而是为了让预测结果更接近真实可交易股票。

如果不做流动性过滤，容易遇到：

1. 想买时买不进去。
2. 想卖时卖不出去。
3. 小资金也可能明显冲击价格。
4. 实际成交价远差于回测价格。
5. 滑点和交易成本被严重低估。
6. 前 10 名容易被低成交额股票占据。

因此正式交易策略必须优先使用带流动性过滤的股票池。

## 5. 当前 label1d 模型训练口径

当前 `label1d` 模型目标：

```text
label_1d__cs_rank
```

模型路径：

```text
outputs/models/20260601_000000__label1d_top40/lightgbm/model.txt
```

训练配置中：

```yaml
train:
  filter_in_universe: true
```

这表示训练样本只使用：

```text
in_universe = true
```

而当前 `in_universe` 包含流动性过滤。

因此结论是：

```text
当前 label1d 模型是考虑了流动性过滤训练出来的。
```

## 6. 为什么取消流动性过滤会导致分数变化

取消流动性过滤后，不只是多了几只股票参与排名。

更重要的是，特征也会变化。

当前特征工程中有大量截面特征，例如：

```text
截面排名特征
截面稳健标准化特征
行业相对特征
```

这些特征都依赖当天股票池。

如果股票池从 3955 只变成 4943 只，那么同一只股票的截面位置会变化，模型输入特征也会变化。

因此：

```text
取消流动性过滤后，分数变化是正常现象。
这不是简单把被过滤股票插回原排名。
```

## 7. 2026-06-08 预测中的实际例子

使用 `20260605` 数据预测 `20260608` 时：

正式口径：

```text
保留流动性过滤
候选池数量：3955
```

诊断口径：

```text
取消流动性过滤
候选池数量：4943
```

新增低流动性股票数量：

```text
988
```

两版共同股票的相关性：

| 指标 | 数值 |
| --- | ---: |
| 分数相关性 | 0.9501 |
| 排名相关性 | 0.9468 |

这说明模型信号整体仍相关，但口径改变足以造成明显排名变化。

无流动性过滤版本的前 50 名中，有 49 只原本不能通过流动性过滤。

这说明低流动性股票会大量挤入前排。

以 `20260605 -> 20260608` 为例：

| 股票 | 代码 | 正式口径排名 | 正式分数 | 取消流动性排名 | 取消流动性分数 |
| --- | --- | ---: | ---: | ---: | ---: |
| 麒麟信安 | 688152.SH | 20 | 0.137795 | 494 | 0.067841 |
| 赛福天 | 603028.SH | 8 | 0.156513 | 554 | 0.064168 |
| 三特索道 | 002159.SZ | 2 | 0.170937 | 331 | 0.079198 |

这些变化不能解释为模型随机波动，而是股票池口径改变造成的特征分布变化。

奥福科技和甘化科工在正式口径下没有分数，是因为没有通过流动性过滤：

| 股票 | 代码 | 20 日成交额均值 | 当日流动性分位 | 取消流动性后排名 |
| --- | --- | ---: | ---: | ---: |
| 奥福科技 | 688021.SH | 59493.0915 | 14.55% | 521 |
| 甘化科工 | 000576.SZ | 71626.1877 | 19.95% | 1711 |

当日 20% 流动性阈值约为：

```text
71846.4486
```

甘化科工接近阈值但仍低于阈值，因此被过滤。

## 9. 正式口径与诊断口径的边界

正式交易使用：

```text
保留流动性过滤
保留 in_universe 训练口径
使用正式 live_predictions 文件
```

诊断分析可以使用：

```text
取消流动性过滤
放宽候选池
检查被排除股票的模型分数
比较不同股票池下的排名变化
```

但诊断口径有明确限制：

1. 不能和正式口径分数直接比较。
2. 不能直接作为正式交易组合。
3. 不能用来证明某只被过滤股票一定值得买。
4. 不能在没有重训和回测的情况下替代正式口径。

## 10. 如果真的想不考虑流动性

如果后续决定研究“不考虑流动性”的策略，不能只在预测时关闭过滤。

必须完整重做：

1. 重新生成无流动性过滤版本的 `universe`。
2. 重新生成特征。
3. 重新生成标签样本。
4. 重新训练 `label1d` 模型。
5. 重新做验证集和测试集回测。
6. 重新加入交易成本、滑点和无法成交约束。
7. 再决定是否可用于交易。

否则会出现：

```text
训练时使用高流动性股票池
预测时使用全量股票池
模型输入分布不一致
分数解释失真
排名不稳定
```

## 11. 后续代码改进建议

为了减少以后误解，后续应在 live 预测管道中补充：

```text
excluded_reason.csv
```

每只未进入候选池的股票都给出明确原因：

| 原因 | 含义 |
| --- | --- |
| `missing_raw_data` | 原始行情或指标缺失 |
| `st_stock` | ST 股票 |
| `north_exchange` | 北交所股票 |
| `too_new` | 上市天数不足 |
| `no_volume` | 当日无成交 |
| `low_liquidity` | 未通过流动性过滤 |
| `missing_features` | 特征构造后缺失 |

同时每日输出中应区分：

```text
rank_tradable
rank_base_no_liq
excluded_reason
```

这样可以同时满足正式交易和诊断分析，避免把两种口径混在一起。
