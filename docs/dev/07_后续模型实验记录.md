## 当前目标

在已经完成 GRU、LightGBM、XGBoost 和简单预测融合实验之后，继续补全以下实验：

1. 树模型稳健性检查：

   - 用这些特征分别重训 LightGBM 和 XGBoost。

3. GRU 与树模型结合的后续方向：
   - 优先考虑用 GBDT 重要性筛出来的特征重训 GRU。
   - 神经网络输出作为树模型额外特征需要 out-of-fold 方式生成训练期预测，避免把 GRU 训练集过拟合泄漏给树模型。
   - 简单预测层加权融合已经跑过，验证集没有稳定增益，不作为主线结果。

### 6. LightGBM residual + MLP 融合实验

新增脚本：

- `src/model_experiments/run_residual_mlp.py`

实验目标：

检验神经网络是否能学习 LightGBM 没解释掉的残差。形式为：

```text
final_pred = lightgbm_pred + alpha * mlp_residual_pred
residual = label_5d__cs_rank - lightgbm_pred
```

这里没有直接用 LightGBM 的训练内残差，而是跑了两组避免 in-sample overfit leakage 的实验。

#### 6.1 Frozen LightGBM + MLP residual

设计：

- Base LightGBM：用 `2016-2020` 训练，并冻结。
- Residual MLP：用冻结 LightGBM 在 `2021-2023` 的样本外预测残差训练。
- valid：`2024`
- test：`2025-2026`

命令：

```bash
python -m src.model_experiments.run_residual_mlp \
  --mode frozen \
  --out-root outputs/models/20260530_235030__residual_mlp \
  --feature-list outputs/models/20260530_205006__feature_selection/features/lightgbm_top40.txt \
  --num-boost-round 800 \
  --early-stopping-rounds 80 \
  --num-threads 16 \
  --log-period 200 \
  --mlp-epochs 6 \
  --mlp-hidden 128 \
  --mlp-dropout 0.1 \
  --mlp-batch-size 8192 \
  --alpha-grid 0 0.25 0.5 0.75 1.0
```

结果：

| split | base IC | base ICIR | residual IC | residual ICIR | best alpha | final IC | final ICIR |
| --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| valid | 0.114013 | 0.787753 | 0.044565 | 0.204464 | 0.25 | 0.120129 | 0.787964 |
| test | 0.095111 | 0.776450 | 0.040240 | 0.301404 | 0.25 | 0.102140 | 0.793801 |

和主 baseline 对比：

| 模型 | 训练/融合口径 | test IC | test ICIR | 说明 |
| --- | --- | ---: | ---: | --- |
| GRU layer1 baseline | 112 特征，GRU layer1 | 0.085123 | 0.648694 | 原深度学习 baseline |
| Frozen LightGBM base | LightGBM top40，只用 2016-2020 训练并冻结 | 0.095111 | 0.776450 | residual 实验里的冻结基模型 |
| Frozen LightGBM + MLP residual | base + `0.25 * residual_mlp` | 0.102140 | 0.793801 | 比冻结基模型和 GRU 强 |
| LightGBM top40 baseline | LightGBM top40，2016-2023 训练 | 0.106769 | 0.817390 | 当前主 GBDT baseline |

结论：

- residual MLP 有弱正信号。
- 加 `alpha=0.25` 能提升 frozen base 的 test IC/ICIR。
- 对比 GRU baseline，Frozen residual 融合更强。
- 但最终 test ICIR `0.793801` 仍低于全量 LightGBM top40 baseline 的 `0.817390`。
- 这说明 residual MLP 能补一点旧 LightGBM 的残差，但不足以超过当前最佳 GBDT baseline。

#### 6.2 OOF/expanding LightGBM + MLP residual

设计：

- 生成 OOF residual：
  - `2016-2020 -> 2021`
  - `2016-2021 -> 2022`
  - `2016-2022 -> 2023`
- Residual MLP：用 2021-2023 的 OOF residual 训练。
- Base valid/test：用 `2016-2023` 训练的 LightGBM top40。
- valid：`2024`
- test：`2025-2026`

命令：

```bash
python -m src.model_experiments.run_residual_mlp \
  --mode oof \
  --out-root outputs/models/20260530_235030__residual_mlp \
  --feature-list outputs/models/20260530_205006__feature_selection/features/lightgbm_top40.txt \
  --num-boost-round 800 \
  --early-stopping-rounds 80 \
  --num-threads 16 \
  --log-period 200 \
  --mlp-epochs 6 \
  --mlp-hidden 128 \
  --mlp-dropout 0.1 \
  --mlp-batch-size 8192 \
  --alpha-grid 0 0.25 0.5 0.75 1.0
```

结果：

| split | base IC | base ICIR | residual IC | residual ICIR | best alpha | final IC | final ICIR |
| --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| valid | 0.128772 | 0.788960 | 0.007722 | 0.039931 | 0.00 | 0.128772 | 0.788960 |
| test | 0.106769 | 0.817390 | 0.004526 | 0.036180 | 0.00 | 0.106769 | 0.817390 |

和主 baseline 对比：

| 模型 | 训练/融合口径 | test IC | test ICIR | 说明 |
| --- | --- | ---: | ---: | --- |
| GRU layer1 baseline | 112 特征，GRU layer1 | 0.085123 | 0.648694 | 原深度学习 baseline |
| LightGBM top40 baseline | LightGBM top40，2016-2023 训练 | 0.106769 | 0.817390 | 当前主 GBDT baseline |
| OOF LightGBM + MLP residual | OOF residual 训练 MLP，best alpha = 0 | 0.106769 | 0.817390 | 没有超过 LightGBM top40 |

结论：

- 在更规范的 OOF stacking 口径下，MLP residual 基本没有可学信号。
- valid/test 上最优 alpha 都是 `0`，也就是不加入 residual。
- 因此当前不建议把 `OOF LightGBM residual + MLP` 作为主结果；它只是在结果上退化为 LightGBM top40 baseline。
- 这也说明 LightGBM top40 的样本外 residual 已经很接近噪声，简单 MLP 难以继续提升。

总体结论：

- Frozen residual 方案能证明“MLP 可以补一点旧模型残差”，但不超过当前最佳 baseline。
- OOF residual 方案更规范，但没有提升。
- 目前不建议继续大规模跑 GRU residual，除非后续想专门证明“时序网络是否比 MLP 更能解释 residual”。从 MLP 结果看，预期收益不高。

| Frozen LGBM + MLP residual | 40 | test | 0.095111 | 0.776450 | 0.040240 | 0.301404 | 0.25 | 0.102140 | 0.793801 |
| Frozen LGBM + MLP residual | 112 | valid | 0.113115 | 0.785695 | 0.043352 | 0.227092 | 0.00 | 0.113115 | 0.785695 |

### 8. LightGBM top40 + MLP 全量特征 residual

上一节“全量 112 特征”同时把 LightGBM base 和 MLP residual 都换成了 112 特征。更合理的融合口径应该是：

```text
LightGBM base: 使用当前最稳的 top40 特征
MLP residual: 使用全量 112 特征 + base_pred
```

也就是说，树模型仍保持当前最强的 top40 设定，神经网络 residual 层再额外读取完整特征，尝试补充树模型没学到的残差。

#### 9.2 Residual-rank MLP

核心思想：

普通 residual MLP 拟合的是数值残差：

```text
residual_value = label_5d__cs_rank - LightGBM_pred
```

这个值很噪声，直接回归容易导致 IC 有一点提升但 ICIR 下降。Residual-rank MLP 改成拟合每天截面内 residual 的排序位置：

```text
residual = label_5d__cs_rank - LightGBM_pred
residual_rank = daily_pct_rank(residual) - 0.5
```

其中：

- `daily_pct_rank(residual)` 在每天截面内取值约为 `[0, 1]`。
- 减去 `0.5` 后，目标变成约 `[-0.5, 0.5]`。
- 正值表示这只股票的真实 label 比 LightGBM 预测更好，LightGBM 低估了它。
- 负值表示 LightGBM 高估了它。

训练：

```text
MLP 输入 = LGB/XGB 的预测、rank、均值、差值等 meta features
MLP 目标 = residual_rank
```

推理：

```text
final_pred = LightGBM_pred + alpha * MLP_predicted_residual_rank
```

这样做的动机：

- 不要求 MLP 精确预测残差数值。
- 只要求 MLP 判断“LightGBM 相对低估/高估”的截面排序方向。
- 对极端残差更稳，理论上比 residual value 更适合 IC/ICIR 目标。

结果：

| alpha | test IC | test ICIR | test top-k return | test rolling return |
| ---: | ---: | ---: | ---: | ---: |
| 0.00 | 0.106769 | 0.817390 | 6.346443 | 5.413240 |
| 0.25 | 0.106720 | 0.816290 | 6.702838 | 5.194393 |
| 0.50 | 0.106665 | 0.815198 | 9.174358 | 5.241071 |
| 0.75 | 0.106605 | 0.814147 | 8.569189 | 5.464956 |
| 1.00 | 0.106537 | 0.813088 | 9.796156 | 5.443183 |

说明：

- valid 上 residual-rank MLP 选到的 best alpha 是 `1.0`。
- test 上如果按 ICIR 排序，最优仍是 `alpha=0`，也就是纯 LightGBM top40。
- 但非零 alpha 的表现非常接近主 baseline：
  - `alpha=0.25` test IC/ICIR 为 `0.106720 / 0.816290`。
  - 和 LightGBM top40 的 `0.106769 / 0.817390` 只差很小。
- residual-rank 是目前最接近主 baseline 的深度融合方法，但还没有稳定超过纯 LightGBM。

  - shallow residual score test IC/ICIR：`0.040240 / 0.301404`
  - deep_ln residual score test IC/ICIR：`0.039146 / 0.289183`
- 但 deep_ln 没有提高这个 residual-value 目标，反而略弱。
- Frozen deep_ln 的 rolling return 比 shallow 更高，但 IC 和 ICIR 都更低，不建议作为主结果。

#### 11.3 residual-value 与 residual-rank 的结论差异

| 方法 | 架构 | 最优 alpha 口径 | test IC | test ICIR | 结论 |
| --- | --- | ---: | ---: | ---: | --- |
| OOF residual-value | shallow | 0.00 by ICIR | 0.106769 | 0.817390 | alpha=0.25 的 IC 更高，但 ICIR 变差 |
| OOF residual-value | deep_ln | 0.00 by ICIR | 0.106769 | 0.817390 | deep_ln 没有帮助 |
| Frozen residual-value | shallow | 0.25 by ICIR | 0.102140 | 0.793801 | 弱于 LGB top40 |
| Frozen residual-value | deep_ln | 0.25 by ICIR | 0.100933 | 0.783880 | 弱于 shallow |
| Residual-rank | deep_ln | 1.00 by ICIR | 0.106824 | 0.820006 | 当前最好深度融合 |

当前判断：

- `residual-rank deep_ln` 的 `alpha=1.0` 是当前离散 alpha 网格 `{0, 0.25, 0.5, 0.75, 1.0}` 上，valid 和 test 都按 ICIR 排序得到的最佳点。
- 这不是连续空间里的严格最优 alpha，只能说明在当前网格中 residual-rank 分支越强越好。
- 原始 residual-value 的 `alpha=0.25` 更像是提高了平均 IC，但同时增加了日度 IC 波动，所以 ICIR 下降。
- 从主指标看，下一步应继续沿 `residual-rank + 稳定 MLP 架构` 改，而不是继续加深 residual-value MLP。
