# 07 后续模型实验记录

日期：2026-05-30

## 当前目标

在已经完成 GRU、LightGBM、XGBoost 和简单预测融合实验之后，继续补全以下实验：

1. 树模型稳健性检查：
   - 按年份拆分 valid/test 的 IC 和 ICIR。
   - 检查策略参数对回测结果的敏感性。
   - 参数包括 top-k 的持仓数量、轮换数量、调仓频率、交易成本，以及 rolling tranche 的持有天数、目标持仓数量、交易成本。

2. 树模型特征选择实验：
   - 使用全量 LightGBM 的 gain importance 选 top20、top40、top60 特征。
   - 用这些特征分别重训 LightGBM 和 XGBoost。
   - 与 112 个全量特征的 LightGBM/XGBoost 对比 IC、ICIR 和回测结果。

3. GRU 与树模型结合的后续方向：
   - 优先考虑用 GBDT 重要性筛出来的特征重训 GRU。
   - 神经网络输出作为树模型额外特征需要 out-of-fold 方式生成训练期预测，避免把 GRU 训练集过拟合泄漏给树模型。
   - 简单预测层加权融合已经跑过，验证集没有稳定增益，不作为主线结果。

### 1. 生成 LightGBM top 特征列表

```bash
python - <<'PY'
import pandas as pd
from pathlib import Path
src=Path('outputs/sdd_gbdt_full/lightgbm/feature_importance.csv')
df=pd.read_csv(src).sort_values('importance_gain', ascending=False, kind='mergesort')
for k in [20,40,60]:

PY
```

### 3. 策略稳健性和参数敏感性

命令：

```bash
python -m src.models.sdd.run_backtest_sensitivity \
  --out-root outputs/sdd_gbdt_sensitivity
```

输出：

- `outputs/sdd_gbdt_sensitivity/lightgbm/valid`
- `outputs/sdd_gbdt_sensitivity/lightgbm/test`
- `outputs/sdd_gbdt_sensitivity/xgboost/valid`
- `outputs/sdd_gbdt_sensitivity/xgboost/test`

关键结论：

- LightGBM test 总体 IC/ICIR：`0.107852 / 0.805562`。
- XGBoost test 总体 IC/ICIR：`0.106861 / 0.794579`。
- test 内部按年份看，最弱年份都是 2026：
  - LightGBM 2026 IC/ICIR：`0.068809 / 0.590046`。
  - XGBoost 2026 IC/ICIR：`0.068036 / 0.575877`。
- 说明 2025 的信号强于 2026 上半年，模型存在阶段性衰减，后续报告里不能只写整体 test。
- 策略参数网格里，`step_days=1`、`cost_bps=0`、较大持仓数量的结果非常高，但这是低成本/高频理想条件下的敏感性结果，不应该作为主报告策略。

### 4. GBDT top 特征重训

使用全量 LightGBM 的 gain importance 选 top20/top40/top60 特征，分别重训 LightGBM 和 XGBoost。

结果汇总：

| 模型 | 特征数 | valid IC | valid ICIR | test IC | test ICIR | test top-k return | test rolling return |
| --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| LightGBM | 112 | 0.125789 | 0.761766 | 0.107852 | 0.805562 | 4.499941 | 3.932256 |
| LightGBM | 20 | 0.128568 | 0.778469 | 0.101924 | 0.727864 | 1.669449 | 2.344338 |
| LightGBM | 40 | 0.128772 | 0.788960 | 0.106769 | 0.817390 | 6.346443 | 5.413240 |
| LightGBM | 60 | 0.125480 | 0.762059 | 0.107416 | 0.798122 | 6.067766 | 4.655122 |
| XGBoost | 112 | 0.125502 | 0.776915 | 0.106861 | 0.794579 | 21.121385 | 6.990298 |
| XGBoost | 20 | 0.126761 | 0.771056 | 0.100933 | 0.723027 | 2.880087 | 2.413259 |
| XGBoost | 40 | 0.128218 | 0.793388 | 0.105494 | 0.804857 | 22.933992 | 8.129507 |
| XGBoost | 60 | 0.125527 | 0.764527 | 0.107205 | 0.795369 | 20.017825 | 6.913186 |

结论：

- top20 特征太少，valid 看起来不差，但 test ICIR 明显下降。
- top40 是当前最稳的特征子集：
  - LightGBM top40 的 test ICIR `0.817390`，高于 112 特征的 `0.805562`。
  - XGBoost top40 的 test ICIR `0.804857`，略高于 112 特征的 `0.794579`。
- top60 接近全特征，但没有 top40 稳定。
- 后续如果报告一个特征选择结果，优先报告 LightGBM top40。

### 5. GRU 使用 GBDT top40 特征

命令：

```bash
python -m src.models.sdd.run_ablation \
  --experiments layer1 \
  --out-root outputs/sdd_gru_gbdt_features_full \
  --processed-dir data/processed \
  --feature-list outputs/sdd_feature_selection/features/lightgbm_top40.txt \
  --custom-name layer1_gbdt_top40
```

训练过程：

| epoch | train loss | valid loss |
| ---: | ---: | ---: |
| 1 | 0.160931 | 0.163202 |
| 2 | 0.159615 | 0.164750 |
| 3 | 0.157913 | 0.168008 |

最佳 epoch 是 `1`。后续 epoch 训练损失继续下降，但验证损失上升，说明过拟合非常快。

和原 GRU layer1 对比：

| 模型 | split | samples | MSE | IC | ICIR | top-k return |
| --- | --- | ---: | ---: | ---: | ---: | ---: |
| GRU layer1 112特征 | valid | 766867 | 0.327678 | 0.095154 | 0.511638 | 0.126012 |
| GRU layer1 top40 | valid | 766867 | 0.326761 | 0.093614 | 0.514228 | 0.175865 |
| GRU layer1 112特征 | test | 1019298 | 0.327920 | 0.085123 | 0.648694 | 0.358659 |
| GRU layer1 top40 | test | 1019298 | 0.327571 | 0.078709 | 0.567075 | 0.840851 |

结论：

- GRU top40 的 valid ICIR 和原模型基本持平，但 test IC/ICIR 明显下降。
- 虽然 top-k 回测收益上升，但主指标 IC/ICIR 变差，因此这组不作为改进。
- 树模型的 top40 特征对 GBDT 有帮助，但不能直接迁移为 GRU 的最佳输入；GRU 可能仍需要时序特征、更多特征冗余或专门的正则化设计。
