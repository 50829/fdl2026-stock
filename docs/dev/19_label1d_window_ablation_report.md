# 19 label1d 窗口消融实验报告

日期：2026-06-09

## 1. 实验目的

本实验回答一个问题：

```text
当前 label1d 模型到底依赖哪些历史窗口？
5 日、10 日、20 日、60 日窗口中，哪些对 IC、策略收益和回撤更重要？
```

这不是重新设计交易策略，而是先做模型输入层面的消融。

## 5. 主要观察

### 5.1 不是只靠 20 日窗口

`no_20d` 删除 20 日窗口后，验证集 ICIR 反而最高：

```text
valid ICIR = 0.580271
```

这说明当前 `label1d` 模型并不是单纯靠 20 日窗口在工作。

但 `no_20d` 的 test rolling Sharpe 低于全窗口基线：

```text
no_20d test rolling Sharpe = 3.554585
all_windows test rolling Sharpe = 3.812269
```

因此不能只因为验证集 ICIR 最高就直接替换线上模型。

### 5.2 全窗口基线仍然是最稳的默认选择

`all_windows` 在验证集和测试集之间更均衡：

```text
valid ICIR = 0.560143
test ICIR = 0.617292
valid rolling Sharpe = 0.995315
test rolling Sharpe = 3.812269
```

它不是每个单项第一，但整体风险收益更均衡。

当前 live 模型暂时不建议替换。

### 5.3 10 日窗口对策略表现很关键

`no_10d` 的 IC 还可以，但 rolling 策略明显恶化：

```text
valid rolling Sharpe = -0.022862
valid 最大回撤 = -0.398101
test rolling Sharpe = 2.219453
test 最大回撤 = -0.181989
```

这说明 10 日窗口可能对持仓路径、换仓稳定性或排序结构有重要作用。

不能只看 IC 来判断窗口重要性。

### 5.4 只用短窗口在测试集看起来强，但验证集偏弱

`short_5_10` 的 test ICIR 最高：

```text
test ICIR = 0.639470
```

但 valid IC 和 valid rolling 表现弱于全窗口：

```text
short_5_10 valid IC = 0.077343
all_windows valid IC = 0.083365
```

这可能说明短窗口在 2025-2026 测试期更适配，但跨年份稳定性还不够。

### 5.5 60 日窗口提供稳定信息

`no_60d` 删除 60 日窗口后，验证集 ICIR 最低：

```text
valid ICIR = 0.524550
```

说明 60 日窗口虽然看起来较慢，但对 `label1d` 这种短周期目标仍可能提供背景趋势或风险状态信息。

## 6. 当前结论

本次消融不支持“label1d 只看 20 天所以不可靠”的判断。

更准确的结论是：

```text
label1d 使用多年训练样本。
输入特征混合了 5、10、20、60 日窗口。
20 日窗口不是唯一核心。
全窗口组合目前仍是更稳的默认 live 模型。
```

当前不建议立即把线上 `label1d_lightgbm_top40_alpha0` 替换成某个消融模型。

如果只按验证集 ICIR，`no_20d` 值得继续研究。

如果只按测试集 ICIR，`short_5_10` 值得继续研究。

但从实盘角度，仍应优先看：

```text
valid/test 一致性
rolling 策略 Sharpe
最大回撤
换手率
成本敏感性
流动性暴露
```

## 7. 下一步

建议继续做三件事：

1. 对 `all_windows`、`no_20d`、`short_5_10` 三组做策略引擎级回测，而不是只看模型内置 rolling 指标。
2. 对三组做成本敏感性矩阵，比较 5、10、20、50 bps 下的收益和回撤。
3. 对三组做 walk-forward 验证，确认不是 2024 或 2025-2026 特定市场环境造成的偶然优势。

优先级：

```text
第一候选：all_windows
研究候选：no_20d
进攻候选：short_5_10
暂不推荐：no_10d、mid_20_60、no_60d
```

## 8. 关于 topk 评估口径的修正

最初窗口消融报告中的 `bt_sharpe` 来自模型训练入口默认配置：

```text
topk = 20
drop = 5
step_days = 5
return_col = label_1d
```

这个口径对 `label1d` 不够合理。

原因是：

```text
label1d 是日频目标。
如果 topk 被理解为每日滚动持仓策略，step_days 应该是 1。
step_days = 5 更像每 5 天抽样一次 1 日收益，不是完整的日频 topk 交易口径。
```

因此重新用已训练好的 7 组预测文件计算：

```text
topk = 20
step_days = 1
drop = 3 和 drop = 5
return_col = label_1d
```

输出文件：

```text
outputs/models/20260609_152416__label1d_window_ablation/topk_step1_metrics.csv
```

### 8.3 修正后的解释

如果按日频 topk 口径：

```text
topk20_drop3：all_windows 的 valid Sharpe 最高，short_5_10 的 test Sharpe 最高。
topk20_drop5：no_20d 的 valid/test Sharpe 都最高，但换手也很高。
```

这会改变前面对窗口组合的判断：

1. `all_windows` 仍然是稳健默认候选，尤其在 `topk20_drop3` 验证集上最强。
2. `no_20d` 在 `topk20_drop5` 口径下非常强，但换手接近 `0.83`，需要成本敏感性验证。
3. `short_5_10` 在测试集很强，但验证集回撤更大，不宜直接替换线上模型。
4. `no_10d` 仍然偏弱，说明 10 日窗口对日频 topk 策略也很重要。

因此当前最准确的结论是：

```text
线上默认模型暂不替换。
下一步重点比较 all_windows、no_20d、short_5_10。
必须用 topk step_days=1 和完整策略引擎做成本敏感性，而不是继续用 step_days=5 的简化 topk 指标。
```
